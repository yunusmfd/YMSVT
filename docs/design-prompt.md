# برومت تصميم واجهات منصة Nova SVT

انسخ هذا البرومت كاملا إلى أداة تصميم UI/UX تعمل بالذكاء الاصطناعي (Figma AI، v0، أو محادثة تصميم مع Claude/GPT) لإنتاج مقترحات تصميم متّسقة لصفحات المنصة.

---

## السياق

صمّم واجهات مستخدم لـ **Nova SVT**، منصة تعليمية ساكنة (بدون تسجيل دخول، بدون خادم خلفي) لمادة "علوم الحياة والأرض" (SVT) الموجَّهة لتلاميذ التعليم الإعدادي والثانوي التأهيلي بالمغرب. المنصة **ثنائية اللغة بالكامل** (عربي RTL / فرنسي LTR) على نفس الرابط، مع تبديل فوري بينهما دون إعادة تحميل الصفحة. الفئة المستهدفة: تلاميذ 11–18 سنة وأساتذتهم، على أجهزة متفاوتة الجودة وشبكات 3G أحيانا — الأداء والوضوح لهما أولوية على الزخرفة.

**قيد صارم**: نظام التصميم أدناه **مُقفَل ونهائي** — لا تقترح ألوانا أو خطوطا جديدة. أي اقتراح يجب أن يستعمل هذه القيَم حرفيا فقط.

### الطباعة
- **عناوين عربية**: Markazi Text (serif)
- **عناوين فرنسية**: Fraunces (serif)
- **نص أساسي عربي**: IBM Plex Sans Arabic
- **نص أساسي فرنسي**: IBM Plex Sans
- **Mono** (أكواد/بيانات صغيرة/eyebrow): IBM Plex Mono
- سلّم الأحجام: 12.5 / 14 / 15 / 16.5 / 18 / 20 / 24 / 28 / 32px، وعنوان Hero بـ`clamp(40px, 6vw, 54px)`

### المسافات والزوايا
- سلّم مسافات: 6 / 10 / 14 / 18 / 22 / 30 / 40 / 56 / 70px
- زوايا: عادية 14px، بطاقات كبيرة 22px، صغيرة 8px، دائرية كاملة (pill) للشارات والأزرار
- عرض الحاوية الأقصى: 1180px
- نقاط الكسر: ≤479px (جوال صغير)، ≤679px (جوال)، ≤899px (تابلت)، ≤1179px (سطح مكتب صغير)

### مكوّنات موجودة يجب إعادة استعمالها لا اختراع بديل لها
بطاقة محتوى (`.card`، صورة 16:9 + شارات + عنوان + وصف)، شارة (`.chip`)، زر أساسي/ثانوي/شفاف (`.btn-primary/.btn-ghost/.btn-glass`)، تبويبات (`.tabs`)، قوائم منسدلة للفلاتر، نافبار بشعار (أيقونة دائرية + "Nova **SVT**") في جهة وعناصر تنقّل مُوسَّطة + أزرار إجراء (بحث/لغة/وضع ليلي/إشعارات/فضاء طالب) في الجهة المقابلة، فوتر داكن ثابت دوما.

## 2. مبادئ UX إلزامية

1. **ثنائية اللغة على نفس الرابط**: صمّم كل شاشة بحالتين (عربي RTL يمينا لليسار، فرنسي LTR)، مع مراعاة انعكاس الأيقونات الاتجاهية (الأسهم) والتخطيط.
2. **الوضع الليلي إلزامي لكل شاشة** — لا تصمّم الوضع الفاتح فقط.
3. **لا شفافية في الشريط العلوي** — خلفية صلبة دائما مهما كانت حالة التمرير.
4. **الصفحة الواحدة ساكنة (Static)** — لا حركات أو انتقالات معقّدة؛ التفاعل الوحيد المتوقّع هو فلترة/تبويب/كشف تدريجي (مثل الاختبار الذي يكشف سؤالا واحدا في كل مرة).
5. **إمكانية الوصول (a11y)**: تباين AA كحد أدنى لكل زوج لون/خلفية، حالة `focus-visible` واضحة، `prefers-reduced-motion` محترمة.
6. **أداء 3G**: تفضيل SVG على الصور الثقيلة حيث أمكن، لا خطوط زخرفية إضافية.

## 3. صفحات المنصة المطلوب تصميمها

صمّم كل صفحة أدناه بحالتيها (فاتح/داكن) وبعرضين (سطح مكتب 1440px، جوال 375px):

### أ. صفحات رئيسية معقّدة (أولوية التصميم)
1. **الصفحة الرئيسية** (`/`) — Hero مركزي بعمود واحد (eyebrow + عنوان بلونين + نص + زرّا CTA)، شبكة 4 بطاقات مزايا بأيقونة ملوّنة، شبكة 3 بطاقات "أحدث الدروس" (صورة + شارة تخصّص في الزاوية + شارات مستوى/مدة + عنوان + وصف)، بطاقة "هل تعلم؟" مميّزة بعلامة اقتباس زخرفية كبيرة وزر مشاركة.
2. **صفحة الدروس** (`/lecons/`) — شريط فلاتر منسدلة (مستوى → مسلك (شرطي) → دورة → وحدة)، عرض الدروس مجمَّعة بعناوين وحدات.
3. **صفحة درس فردي** — شريط تقدّم قراءة، شريط معلومات، فهرس محاور جانبي Sticky، محاور متنوعة (نص/صورة/فيديو/أطر تربوية بـ7 أنماط بصرية موحّدة/جدول/رابط معجمي)، اختبار QCM يكشف سؤالا فسؤالا مع تصحيح فوري، تنقّل سابق/تالي.
4. **الفروض والامتحانات** (`/devoirs-examens/`) — تبويبان (فروض بفلاتر / امتحانات بتبويبات فرعية محلي-جهوي-وطني حسب المستوى الإشهادي).
5. **مجلّة الموسوعة** (`/encyclopedie/`) — تصميم "غلاف مجلة": شبكة 11 قسما مرقّمة (01–11) بأيقونة/إيموجي لكل قسم ووصف سطر واحد.
6. **فضاء المراجعة** (`/revision/`) — 3 أقسام رئيسية (مراجعة الدروس بنفس فلاتر صفحة الدروس / الاستعداد للفروض / الاستعداد للامتحانات).

### ب. صفحات قوائم بنمط موحّد (صمّم نمطا واحدا قابلا للتكرار)
`/encyclopedie/articles/`، `/scientifiques/`، `/decouvertes/`، `/glossaire/`، `/chronologies/`، `/saviez-vous/`، `/galerie/`، `/organismes/`، `/roches-mineraux/`، `/geologie-maroc/`، `/experiences-historiques/`، `/labo-virtuel/`، `/blog/` — كلها شبكات بطاقات بسيطة، بعضها بفلاتر تبويب بسيطة (نوع التجربة في المختبر مثلا).

### ج. صفحات تفاصيل بنمط موحّد (صمّم نمطا واحدا قابلا للتكرار)
تفاصيل عنصر موسوعة (عالِم/اكتشاف/كائن حي/صخرة/موقع جيولوجي/تجربة تاريخية) — عنوان، صورة إن وُجدت، وصف، بيانات وصفية (تاريخ/تصنيف)، مسار تنقّل (breadcrumb).

### د. صفحات ثانوية (تصميم مبسّط)
فضاء الطالب (`/mon-espace/` — تقدّم محفوظ محليا فقط، بلا حساب)، عن المنصة، اتصل بنا (نموذج بسيط)، سياسة الخصوصية، صفحة 404 (رسالة + زرّا عودة).

## 4. المُخرَجات المطلوبة

لكل صفحة من القسم "أ" (الأولوية):
- Wireframe أو Mockup عالي الدقّة بالحالتين (فاتح/داكن) وبعرضين (1440px/375px).
- توضيح أي تفاعل ديناميكي (فلترة، كشف تدريجي، تبويب).
- التزام حرفي بقيَم الألوان/الخطوط/المسافات أعلاه — أي انحراف عنها يُعتبر خطأ تصميم.

لصفحات القسم "ب" و"ج": مخطّط نمط بطاقة واحد قابل لإعادة الاستعمال بدل تصميم كل صفحة منفردة.
